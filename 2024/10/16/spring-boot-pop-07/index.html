Spring Boot project-oriented programming（7）使用 MariaDB 打造RESTful API專案：從安裝到 CRUD 教學

前言

我們先前的專案使用 H2 資料庫，但遇到了一個重大問題：專案關閉後，資料庫中的資料就會消失。

因此，本次我們將改用需要安裝的資料庫，這樣資料能長期保存，即使關機也不會消失，只要我們不主動刪除，就能永久保存。

談到資料庫，許多人會想到 MySQL，因為它曾與 Java 一樣，隸屬於 Oracle。然而，我們這次的專案並不會使用 MySQL。

為什麼避開 MySQL？

回顧過去，Oracle 曾免費提供 Oracle JDK 給所有人與企業使用，但某天突然宣布 Oracle JDK 改為收費，不付費就會收到律師警告信。雖然 Oracle 另外提供了免費的 OpenJDK，但這個轉變導致許多企業改用自己的 OpenJDK，避免被收費。

由此可見，若選用 MySQL，未來也有可能會遭遇類似情況。為了避免這種風險，我們將改用 MariaDB。

MariaDB 安裝與建立資料庫

MariaDB 是一個開源且可免費商用的資料庫，由部分原 MySQL 開發團隊創建。它與 MySQL 高度相容，對開發者來說幾乎沒有學習障礙。
