Spring Boot project-oriented programming（6）API專案 測試

專案測試與部署指南

功能測試

專案已完成，接下來需測試各個功能以確保正常運作。
注意：

  • 每次專案重新啟動時，資料庫的內容將會重置。
  • 若使用 Spring Boot DevTools，修改並儲存程式碼後會自動重啟專案，同時清除資料。

使用 Hoppscotch 測試 RESTful API

我們將使用 Hoppscotch 測試 API 功能。相比於需要登入的 Postman，我更推薦 Hoppscotch，因其更簡單直接。

  1. Hoppscotch 網頁版
  2. Chrome 擴充套件

為了方便測試，這裡提供一個預先設定好的檔案供下載：API 測試範例下載。

匯入範例設定

  1. 在 Hoppscotch 左側點擊 資料夾圖示。

  2. 選擇 Import from Hoppscotch。

  3. 匯入下載的 JSON 檔案後，按 Import。

  4. 你將會看到 todo 資料夾，點開即可查看 API 測試範例。

API 測試操作

1. Create Todo

  • 點選 Create todo，按 Send 即可新增 todo。
  • 在 Body 區域可看到 title 和 completed，你可以嘗試修改後再發送觀察變化。

2. Get All Todos

  • 切換到 Get all todos，按 Send 即可取得所有 todo。若先前多次新增，這裡會顯示多筆資料。

3. Get Todo by ID

  • 點選 Get todo by id 並按 Send，可取得指定 ID 的 todo。

4. Update Todo

  • 在 Update todo 中修改資料，按 Send 可更新 todo 的內容。

5. Set Todo Completed / Uncompleted

  • 使用 Set todo completed 和 Set todo uncompleted 來變更狀態。

6. Delete Todo by ID

  • 在 Delete todo by id 中按 Send，可刪除指定 ID 的 todo。

專案打包成 JAR 檔

為了部署方便，我們將專案打包成 JAR 檔，避免使用 GUI 介面和減少 RAM 佔用。這對於 1GB RAM VPS 特別重要，因為在低資源環境下，無法有效使用像 IDEA 這樣的開發工具。

打包流程

  1. 在 IDEA 右側選擇 m 並展開 restfulapi → Plugins → clean。

  2. 連按兩次 clean.clean 清理舊的 target 資料。

  3. 展開 Lifecycle，連按兩次 package 打包成 JAR 檔案。

JAR 檔會出現在專案的 target 資料夾中： 

target/restfulapi-0.0.1-SNAPSHOT.jar

啟動 JAR 檔

在命令提示字元中進入 target 資料夾後輸入： 

java -jar restfulapi-0.0.1-SNAPSHOT.jar

將 JAR 檔上傳至 Linux VPS 伺服器後，也可用相同指令啟動。

在 Linux 安裝 JDK

以下的方式可在 Debian 和 Redhat 系統上安裝 Amazon Corretto JDK：

  1. 下載 JDK 

    wget https://corretto.aws/downloads/latest/amazon-corretto-17-x64-linux-jdk.tar.gz

  2. 解壓縮檔案 

    tar -zxvf amazon-corretto-17-x64-linux-jdk.tar.gz

  3. 移動至 /usr/share 目錄 

    sudo mv amazon-corretto-17.0.12.7.1-linux-x64 /usr/share

  4. 設定環境變數
    編輯 ~/.bashrc： 

    sudo nano ~/.bashrc

    在最後新增以下兩行： 

    export JAVA_HOME=/usr/share/amazon-corretto-17.0.12.7.1-linux-x64
export PATH=$JAVA_HOME/bin:$PATH
    • 按 Ctrl+O 儲存並按 Enter 確認。
    • 按 Ctrl+X 退出編輯器。

  5. 啟用環境變數 

    source ~/.bashrc

  6. 啟動專案 

    java -jar restfulapi-0.0.1-SNAPSHOT.jar

透過上述步驟，你就能順利測試和部署專案，並在 VPS 上運行應用程式。
